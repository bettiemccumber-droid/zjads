# ZJADS 产品需求文档（PRD）

| 项目 | 内容 |
|------|------|
| 产品名称 | ZJADS — 联盟数据采集与广告投放分析中台 |
| 版本 | v1.0 |
| 目标用户 | 公司内部广告运营（员工）、老板/经理（管理员） |
| 数据库 | MySQL 8 |
| 参考产品 | [联盟营销数据采集系统](https://affiliate-marketing-saas.shop/)、[KYADS](https://kyads.net/) |
| 业务参考 | 现有 affiliate 项目（仅借鉴规则，不抄代码） |

---

## 1. 产品定位

ZJADS 面向**公司内部**联盟投放团队，解决：

1. **多联盟平台、多 Channel、多 Token** 下订单数据采集与标准化；
2. **Google Ads 花费**（徐版脚本 → Google Sheet）与联盟佣金按商家/Campaign 对齐；
3. **ROI 与经营看板**（与日常看数口径一致）；
4. **佣金失效/拒绝预警**（确认周期 1～6 个月，主 ROI 含待确认佣金）。

**非目标（v1）**：对外多租户 SaaS、开放注册、提现全流程（可二期从旧项目迁移）。

## 2. 用户与权限

### 2.1 角色

| 角色 | 代码 | 能力 |
|------|------|------|
| 管理员 | `admin` | 查看全公司数据、公司经营看板、全部员工告警；管理用户；查看全部渠道账号（Token 脱敏） |
| 运营 | `operator` | 仅自己的渠道账号、订单、Sheet、报表；按平台添加/维护自己的 API Token |
| 只读 | `viewer` | 同运营但不可改 Token、不可触发采集 |

### 2.2 数据隔离

- 所有业务数据带 **`owner_user_id`**（渠道账号创建者）。
- 非 `admin` 请求：**强制过滤** `owner_user_id = 当前用户`。
- `admin`：支持 `view=company`（公司合计）与 `view=employee&userId=`（下钻个人）。

### 2.3 账号管理

- 由**管理员**创建员工账号（邮箱 + 初始密码），不做公开注册。
- 员工修改自己的用户名、密码。

## 3. 核心业务概念

### 3.1 Channel（渠道）

联盟后台中的渠道标识（如 PartnerMatic 的 `PM08026591`），与 Google Ads「广告渠道类型」无关。

- **一个采集单元** = `channel_account` = 平台 + Channel ID（可选）+ API Token + 联盟序号（如 `pm1`）。
- 同一员工：同一平台 + 同一 Channel **不可重复**（唯一约束）。
- 不同员工可配置相同 Channel（各自投放）；管理员界面**提示疑似重复**，不阻止。

### 3.2 联盟序号（affiliate_alias）

写在 Google Ads **广告系列名**中（如 `596-pm1-商家-US-0826-71017`），用于：

- 广告花费与联盟订单归因；
- 与 `channel_account.affiliate_alias` 对齐。

### 3.3 订单状态标准化

各平台原始状态不同，入库时：

- 保留 `raw_status`；
- 映射为 `normalized_status`：`approved` | `pending` | `rejected` | `unknown`；
- 映射表 **`platform_status_mappings`** 可配置，新增平台只加映射行。

**失效/拒绝**（用于预警）：`REJECTED`、`EXPIRED`、`CANCELED` 等归入 `rejected`。

### 3.4 佣金确认周期

- 商家佣金确认常需 **1～6 个月**。
- **主 ROI 必须使用「总佣金」（含待确认）**，不可用「仅已确认」作为主 KPI（否则近期数据失真）。
- 结算页展示：已确认 / 待确认 / 已拒绝分列。

## 4. 功能需求

### 4.1 P0 — 必须（第一期）

#### F1 认证

- 登录 / 登出（JWT）。
- 当前用户信息、角色。

#### F2 我的平台账号（按平台）

- 平台列表动态加载（不限平台数量）。
- 按平台 Tab/卡片：**添加渠道账号**。
  - 平台、Channel ID（PM 等必填）、API Token、联盟序号、显示名称。
- 列表、编辑、停用、删除（软删或 `is_active`）。
- Token 加密存储；界面仅显示 `****` + 后四位。

#### F3 联盟订单采集

- 选择日期范围 → 触发采集（仅自己的 `channel_account`）。
- 队列异步：每个渠道账号一个子任务。
- 首期平台：**PartnerMatic**；架构支持扩展 Adapter。
- 任务状态：成功/失败条数、错误信息、日志。

#### F4 结算查询

- 快捷日期（昨天、最近 7 天、本月等）。
- 视图：**商家汇总** / **订单明细**。
- 汇总字段：订单数、订单金额、总佣金、已确认、待确认、已拒绝、结算率、拒付率。
- 筛选：平台账号、订单状态、商家 ID/名称、订单 ID 等。

#### F5 数据采集（商家汇总 + ROI）

- 与 affiliate SaaS 主表对齐的指标：
  - 订单总数、总预算、**总佣金**、总广告费、**整体 ROI**。
- 商家表：排名、广告系列、商家 ID、预算、展示、点击、广告费、订单数、总佣金、CR、EPC、CPC、**ROI**。
- **ROI 公式**：`(总佣金 − 广告费) ÷ 广告费`（总佣金含待确认、已拒绝等所有订单佣金 SUM）。
- 导出 Excel（商家汇总）。

#### F6 佣金失效预警

- 规则（每用户）：启用、统计窗口（天）、失效佣金金额阈值、失效率阈值（%）。
- 按 **商家 ID** 在窗口内统计；触发后生成 `open` 告警。
- 采集完成或「立即检查」后执行。
- 展示：商家名、失效佣金、失效率、触发原因；支持确认(ack)。
- 管理员：查看全公司未处理告警 + 汇总金额。

#### F7 管理员 — 经营看板

- 日期范围。
- **公司整体**：总佣金、总广告费、利润（佣金−广告费）、整体 ROI、订单数；在途待确认 / 已确认 / 失效佣金辅助数字；未处理告警数。
- **按员工对比表**：每人佣金、广告费、ROI、失效佣金、告警数。
- **下钻**：进入某员工与本人一致的商家汇总/结算/告警。

#### F8 用户管理（管理员）

- 创建/停用员工、分配角色。

---

### 4.2 P1 — 重要（第二期）

#### F9 Google 表格（广告数据）

- 登记 Sheet URL，主表 **`raw_daily_report`**（徐版脚本输出）。
- 拉取 CSV/API → 解析英文表头 → 聚合 **`ad_campaign_daily`**。
- `campaign_name` 解析规则 → `affiliate_alias`、`merchant_id`。

#### F10 第二联盟平台 Adapter

- LinkBux 或 LinkHaitao（二选一优先）。

#### F11 推荐榜 / 稳定榜 Top10

- 规则：ROI 阈值、推广人数等（可配置）。

#### F12 Campaign 分析（KYADS 向）

- Campaign 级 ROAS、花费/佣金对比。

#### F13 审计日志

- 管理员操作、Token 变更记录。

---

### 4.3 P2 — 优化（第三期）

- 更多联盟平台 Adapter；
- 关键词/搜索字词表（徐版 `raw_daily_keywords` 等）；
- 规则型投放建议；
- 提现维度（可提现/提现中/已提现）；
- 定时采集 Cron。

## 5. 指标口径（定稿）

| 指标 | 公式 | 说明 |
|------|------|------|
| 总佣金 | `SUM(commission)` 日期内所有订单 | 主 ROI 分子部分，**含待确认** |
| 总广告费 | `SUM(cost)` 归因维度内广告日数据 | 统一 USD |
| ROI | `(总佣金 − 总广告费) / 总广告费` | 广告费=0 时显示 0 或 — |
| CR | `订单数 / 点击数 × 100%` | |
| EPC | `总佣金 / 点击数` | |
| CPC | `广告费 / 点击数` | |
| 失效率 | `失效佣金 / 窗口内总佣金 × 100%` | 预警用 |
| 公司 ROI | 各员工加总后的总佣金与总广告费代入上式 | 与 personal 公式一致 |

## 7. 界面结构（草案）

```
登录
├── 运营工作台（员工）
│   ├── 数据采集（商家汇总 + ROI）
│   ├── 结算查询
│   ├── 我的平台账号（按平台）
│   ├── Google 表格（P1）
│   └── 个人设置
└── 管理后台（admin 额外）
    ├── 经营看板（公司 + 按员工 + 下钻）
    ├── 失效佣金告警（全公司）
    ├── 员工管理
    └── （各模块支持按员工筛选）
```

## 8. 验收标准（P0）

1. 员工 A 只能看到 A 的渠道账号与订单；员工 B 看不到 A 的数据。
2. 管理员可选择日期，看到公司 ROI 与员工对比表，并可下钻 A 的数据与 A 本人一致。
3. PartnerMatic：配置 Channel + Token 后，采集日期范围内订单入库，`normalized_status` 正确。
4. 商家汇总 ROI 与手工按公式计算一致（总佣金含 pending）。
5. 失效佣金超过阈值时产生告警；管理员可见全公司 open 告警列表。

## 10. 附录：广告系列命名规范

```
编号-联盟序号-商家名-国家-日期-商家数字ID
示例：596-pm1-Champion-US-0826-71017
```

- 联盟序号：第 2 段（`pm1`）。
- 商家 ID：从右向左优先 ≥5 位纯数字段。
- PB 商品：广告组名含 ASIN（10 位），用于商品级匹配（P1/P2）。
